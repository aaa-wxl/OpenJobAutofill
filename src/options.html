OpenJobAutofill

维护本机简历资料

数据只保存在本机的 chrome.storage.local，不会走同步云端。AI 只分析页面结构和本地资料字段目录，不会读取你的简历值。点击开始填写后会直接写入高置信度字段，低置信度字段可在右侧资料栏查看和手动复制。

首次使用

三步开始网申填写

  1. 填写资料 先在下面按模块维护本机简历资料。
  2. 保存资料 点击底部“保存资料”，看到浮层提示后就已保存到本机。
  3. 开始填写 打开招聘表单页，点击插件图标里的“开始填写”。

AI 页面分析是可选增强：如果希望模型辅助理解表单字段，再到下方 API 设置填写 API Key、Base URL 和模型名，并点击“测试连接”。不配置 API 时仍可使用本地规则填写。

资料目录

简历资料

这里按通用网申栏目拆开填写。保存后资料只写入本机扩展存储，需要迁移或备份时再导出资料备份。

资料加载后，编辑任意字段会在这里提示是否需要保存。

填写提示
隐私边界

资料只保存在本机 chrome.storage.local。

AI 只看页面字段和资料字段目录，不接收具体资料值。

插件不会自动提交表单。

API 设置（可选，用于 AI 页面分析）

可用变量：{{modelJson}}、{{messagesJson}}、{{systemPromptJson}}、{{userPromptJson}}、{{profileJson}}、{{profileCatalogJson}}、{{fieldsJson}}、{{scanJson}}。其中 profile* 变量都是本地字段目录，不含具体值。
